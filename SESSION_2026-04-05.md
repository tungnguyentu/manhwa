# Session Summary — 2026-04-05

## Work Done

### 1. hentaivnx.com Support (Chrome Extension)
- Added `hentaivnx.com` and `*.hentaivnx.com` to `extension/manifest.json` `host_permissions` and `content_scripts.matches`
- Rewrote `collectPanelImageUrls()` in `extension/content.js` to check 8 lazy-load attributes (`data-lazy`, `data-wpfc-original`, `data-url`, etc.) and accept CDN URLs without file extensions
- Fixed `detectSourceLang()` to return `"ja"` for hentaivnx.com
- Added scroll-based lazy-load trigger in `waitForPanelImages()` and debug logging

### 2. Scraper Fix — Cloudflare Bypass
**Root cause**: hentaivnx.com resets connections from headless browsers. Non-headless Playwright works after visiting the homepage first (acquires Cloudflare session cookies).

**Fix in `toon/scraper/downloader.py`**:
- `_get_image_urls_via_playwright_mcp()` now visits the site homepage before navigating to the chapter URL
- Tries `headless=False` first (bypasses Cloudflare), falls back to `headless=True` then Firefox
- Updated `_EXTRACT_IMAGES_JS` to collect `data-src`, `data-lazy`, `data-original` etc. alongside `src`

**Image structure on hentaivnx.com**:
- CDN: `sv4.2tcdn.cfd`
- Pattern: `https://sv4.2tcdn.cfd/{series-slug}/{chapter-num}/{page}.jpg`
- Most chapter images are ~950×19000px tall strips (lazy-loaded via `data-src`)
- Logo image `hentaivnx.com/images/logo-hentai.png` filtered by `_filter_panel_images()`

### 3. Learn Pipeline — FastAPI Endpoint
Added `/api/learn` endpoint to `toon/api.py`:
- Accepts `{ url, image_urls, series_slug, force_reextract }`
- Runs: scrape → extract (skip_attribution) → build_profiles → learn_style
- Background task with SSE progress via existing `_sse_queues`

Added `LEARN_CHAPTER` message handler to `extension/background.js` and `triggerLearn()` function to `extension/content.js` with a green "📚 Learn" button in the sidebar header.

### 4. OCR Speed Fix (55 min → ~2 min for 6 chapters)
**Root cause**: Each chapter had ~5 GLM-5 attribution API calls (speaker attribution) taking 1-3 min each. 6 chapters × 5 calls × 2 min = 60 min.

**Fix**: Added `skip_attribution=True` parameter to `extract_chapter()` in `toon/extractor/vision.py`:
- When `True`: saves all OCR results directly as `UNKNOWN` speaker, zero API calls
- When `False` (default): full GLM-5 attribution as before

Learn flow (`webui.py` + `api.py`) now passes `skip_attribution=True`.
Translate flow unchanged — still does full attribution (speaker names needed for voice-consistent translation).

Also increased `OCR_CONCURRENCY` from 4 → 8 (Apple Neural Engine handles it well).
